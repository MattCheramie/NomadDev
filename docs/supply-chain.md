# Supply chain verification

Every NomadDev release ships three classes of supply-chain artifact
alongside the binary / image. This page documents what they are and
how an operator verifies them before deploying.

## What ships per release

For each binary (`orchestrator-linux-amd64`, `orchestrator-linux-arm64`):

| File | Purpose |
|---|---|
| `orchestrator-linux-<arch>` | The binary itself. |
| `orchestrator-linux-<arch>.sha256` | SHA-256 checksum. |
| `orchestrator-linux-<arch>.spdx.json` | SBOM in SPDX-JSON format (generated by Syft). |
| `orchestrator-linux-<arch>.sig` | Cosign signature (keyless, Sigstore Fulcio + Rekor). |
| `orchestrator-linux-<arch>.pem` | Signing certificate. |

For the container image at `ghcr.io/mattcheramie/nomaddev:<tag>`:

- **Image signature** — `cosign sign` against the manifest digest, keyless.
- **SBOM attestation** — `cosign attest --type spdxjson`, SPDX-JSON predicate.
- **Multi-arch manifest** covering `linux/amd64` and `linux/arm64`.

## Verify a binary

```sh
TAG=v1.2.3   # the release tag you downloaded
ARCH=amd64   # or arm64

# 1) Checksum.
sha256sum -c orchestrator-linux-${ARCH}.sha256

# 2) Cosign signature (keyless — no public key, identity comes from
#    the OIDC cert chained to Sigstore's Fulcio root and logged in
#    Rekor at sign time).
cosign verify-blob \
    --signature orchestrator-linux-${ARCH}.sig \
    --certificate orchestrator-linux-${ARCH}.pem \
    --certificate-identity-regexp 'https://github.com/MattCheramie/NomadDev/\.github/workflows/release\.yml@refs/tags/.*' \
    --certificate-oidc-issuer https://token.actions.githubusercontent.com \
    orchestrator-linux-${ARCH}
```

If the certificate-identity-regexp doesn't match, the binary wasn't
built by this repo's release workflow on a tag push — refuse to run it.

## Verify the container image

```sh
TAG=v1.2.3

# Resolve the immutable digest (don't sign by tag; tags move).
DIGEST=$(docker buildx imagetools inspect \
    ghcr.io/mattcheramie/nomaddev:${TAG} --format '{{.Manifest.Digest}}')

# 1) Signature.
cosign verify \
    ghcr.io/mattcheramie/nomaddev@${DIGEST} \
    --certificate-identity-regexp 'https://github.com/MattCheramie/NomadDev/\.github/workflows/release\.yml@refs/tags/.*' \
    --certificate-oidc-issuer https://token.actions.githubusercontent.com

# 2) SBOM attestation.
cosign verify-attestation \
    ghcr.io/mattcheramie/nomaddev@${DIGEST} \
    --type spdxjson \
    --certificate-identity-regexp 'https://github.com/MattCheramie/NomadDev/\.github/workflows/release\.yml@refs/tags/.*' \
    --certificate-oidc-issuer https://token.actions.githubusercontent.com
```

Run by **immutable digest**, not by tag, in `docker compose` /
`quickstart-docker.sh` once you've verified — that closes the same
tag-race the sandbox runner closes for `alpine:3.20` (see Phase 8.2 in
[`README.md`](../README.md)).

## CI-side guards

Two CI jobs (defined in
[`.github/workflows/ci.yml`](../.github/workflows/ci.yml)) run on every
PR and on every push to `main`:

- **Trivy** scans the production Dockerfile build and fails on any
  `HIGH` or `CRITICAL` CVE in the OS or Go module layer
  (`ignore-unfixed: true` so we don't block on vulnerabilities that
  upstream hasn't patched yet — those still surface in the SBOM for
  downstream operator visibility).
- **govulncheck** scans the Go module graph against
  `golang.org/x/vuln/db` and fails on any vuln that's actually
  reachable from the orchestrator's call graph.

A release tag won't ship if either job is red — the `release` workflow
won't fire on a commit that doesn't pass `main`'s required checks.
